# Math Scratch 使用手册（Help）

> 面向日常数学推导、整理、复盘与 AI 辅助求解的前端工作台。

## 1. 项目定位

Math Scratch 不是“单条公式输入器”，而是“推导过程工作台”。

它的核心目标是：

1. 把推导过程拆成可管理的行、页、草稿本。
2. 让推导过程可以保存、回溯、恢复，而不是一次性编辑。
3. 在本地完成数据持久化与备份，不依赖自建后端。
4. 让 AI 基于你的真实草稿上下文继续推导。

## 2. 适用场景

1. 课堂/自学做题：按“每步一行”记录，便于检查逻辑。
2. 科研/作业整理：将同一主题拆成多页，逐步归档。
3. 多分支尝试：保存快照后尝试不同路径，随时回退。
4. AI 协作：把当前推导上下文喂给 AI，减少重复描述题目。
5. 结果输出：导出本册、导出组合包、导出全量备份。

## 3. 快速开始

1. Node.js 18+
2. npm 9+

```bash

```


## 4. 界面总览

页面分为三块：

1. 左侧工作区：草稿本与历史记录（快照）。
2. 中间编辑区：分页、逐行编辑、工具条、输出预览。
3. 右侧 AI 区：`AI 助手` 与 `设置` 两个页签（会话、请求设置、提示词、上下文控制）。

## 5. 核心功能怎么用

## 5.1 草稿本与分页

1. 新建草稿本：在左侧输入名称并点击“新建”。
2. 选择草稿本：点击列表项切换。
3. 重命名草稿本：编辑器顶部标题输入框失焦或回车保存。
4. 删除草稿本：点击草稿本右侧垃圾桶，进入回收站。
5. 分页管理：中间页签栏可切页、加页、删页（至少保留 1 页）。

提示：

1. 删除草稿本后若列表为空，会自动创建一个默认草稿本，避免工作区空状态。
2. 切页时会自动载入该页行内容并聚焦首行。

## 5.2 行级编辑

每一行是一个 `math-field`，支持独立编辑、复制、删除。

行操作：

1. `+`：在当前行后插入空行。
2. `=`：在当前行后插入仅含 `=` 的新行。
3. 垃圾桶：删除本行。
4. 复制图标：复制该行 LaTeX。

多行选择：

1. 点击行号可选中/取消选中单行。
2. `Shift + 点击行号` 可做区间多选。
3. 选中后出现批量工具栏，可复制或批量删除。

快捷键：

1. `Enter`：若当前由 MathLive 处理补全/候选，则保持 MathLive 自身行为；否则插入新行，并按“新行首字符为等号”设置决定是否预填 `=`。
2. `Shift/Ctrl/Cmd + Enter`：不再绑定额外快捷换行行为。

## 5.3 快捷模板

工具条提供常用模板按钮，点击会插入到当前激活行。

常见模板包括：

1. `\frac{}{}`
2. `^{}`
3. `_{}`
4. `\sqrt{}`
5. `\sum_{n=1}^{\infty}`
6. `\int_{a}^{b}`
7. `\Rightarrow`
8. `\approx`

## 5.4 输出区（渲染预览）

输出区有三种模式，影响“复制输出文本”和“渲染内容来源”：

1. `当前行 (current)`：输出当前激活行。
2. `全部行 (all)`：复制时按多行原样换行拼接；渲染时会按 Aligned 预览。
3. `Aligned`：输出为 `\begin{aligned}...\end{aligned}`。

你可以：

1. 一键复制输出区文本。
2. 收起/展开输出区。

## 5.5 历史记录（快照）

快照用于记录“某一时刻当前页的完整行内容”。

操作：

1. 保存：填写可选名称后点击“保存”，或用 `Ctrl/Cmd + S`。
2. 搜索：按名称和公式内容关键字筛选。
3. 恢复：将快照内容载入当前页。
4. 重命名：双击记录名或点击“重命名”。
5. 删除：单条垃圾桶删除，或勾选后批量删除。
6. 复制：复制快照为 Aligned。

限制：

1. 每个草稿本最多保留 200 条快照，新快照会插入最前，超限时丢弃最旧条目。

## 5.6 回收站

草稿本删除、快照删除都先进入回收站，不是立即永久删除。

支持：

1. 恢复单条。
2. 全部恢复。
3. 清空回收站。

恢复规则：

1. 恢复草稿本时会重新生成 ID，避免和现有冲突。
2. 恢复快照时若原草稿本不存在，会自动创建一个“恢复草稿本”承载该快照。
3. 名称冲突时会自动追加“(恢复)”或编号。

限制：

1. 回收站最多保留 400 条，超限后丢弃最旧记录。

## 5.7 导入导出

支持三种导出：

1. 导出本册：当前草稿本完整 JSON。
2. 组合导出：当前草稿本 + 快照（可只导出已选快照）。
3. 导出全部备份：所有草稿本 JSON。

导入规则：

1. 仅支持 JSON。
2. 导入采用“追加”策略，不覆盖现有草稿本。
3. 重名会自动加后缀（如“(导入)”）并可继续编号。

## 6. AI 助手完整说明

## 6.1 会话区（AI 助手页）

1. 新建会话：创建空会话。
2. 会话切换：点击左侧会话历史。
3. 删除会话：垃圾桶删除。
4. 会话内消息：
   1. 用户消息支持“重试”（从该提问重新生成后续）。
   2. 任意消息支持编辑、删除。

发送规则：

1. 输入框 `Enter` 发送。
2. `Shift + Enter` 换行。
3. 请求进行中会禁用发送按钮。

## 6.2 API 节点设置（设置页）

每个节点包含：

1. 节点名称
2. `baseUrl`
3. `model`
4. `apiKey`（可选）
5. `customParams`（JSON 对象）

URL 拼接规则：

1. 以 `/chat/completions` 结尾：直接使用。
2. 以 `/v{数字}` 结尾：自动追加 `/chat/completions`。
3. 其他情况：自动追加 `/v1/chat/completions`。

示例：

1. `https://api.openai.com/v1` -> `https://api.openai.com/v1/chat/completions`
2. `https://xxx.com/openai/v1` -> `https://xxx.com/openai/v1/chat/completions`
3. `https://xxx.com/chat/completions` -> 保持不变

`customParams` 规则：

1. 必须是 JSON 对象，例如：`{"temperature":0.2,"max_tokens":2048}`。
2. 如果 JSON 非法或不是对象，请求会被拦截并提示错误。
3. 这些参数会并入请求体。

## 6.3 系统提示词

支持多条系统提示词模板：

1. 新增提示词。
2. 选择当前提示词。
3. 编辑名称和内容。
4. 删除提示词（至少保留一条）。

请求时会把“当前提示词”作为 system message 注入。

## 6.4 思考模式（`off / on / deep`）实际效果

### `off`

1. 请求体不注入 `enable_thinking`。
2. 系统提示词不额外追加“思考要求”。

### `on`

1. 请求体注入 `enable_thinking: true`。
2. 系统提示词追加“请在回答前先进行必要思考”。

### `deep`

1. 请求体同样注入 `enable_thinking: true`。
2. 系统提示词追加更强约束：“请启用更深入的思考，优先给出完整推导”。

说明：

1. `on` 与 `deep` 的协议层都开启 `enable_thinking`，差异主要来自提示词强度。

## 6.5 上下文来源（`none / current-flow / selected-lines / whole-notebook`）实际效果

请求发送前会按该选项生成“额外 system 上下文消息”。

### `none`

1. 不注入草稿上下文。
2. AI 只看系统提示词 + 会话历史。

适用：

1. 做纯概念问答，不希望被当前草稿干扰。

### `current-flow`

1. 读取“当前页全部行”。
2. 自动组装为 `aligned` 块后注入上下文。

适用：

1. 让 AI 延续当前页完整推导。

### `selected-lines`

1. 只读取你当前选中的多行（按行号选择）。
2. 同样转成 `aligned` 注入。
3. 若当前没有选中行，请求会被阻止并提示错误。

适用：

1. 仅针对某几步做局部修正，不想喂整页内容。

### `whole-notebook`

1. 读取当前草稿本的所有分页。
2. 每页独立组装为 `页面名 + aligned`，再拼接注入。

适用：

1. 跨页引用前置结论，做全局一致性检查。

## 6.6 流式接收与渲染

AI 默认走 `stream: true`。

行为：

1. 前端持续消费 SSE 增量，边收边更新 assistant 消息。
2. 同时支持多种字段格式：`delta.content`、`reasoning_content`、`output_text_delta` 等。
3. 如果流式分支没有得到可渲染内容，会回退解析普通 JSON 响应。

思考内容显示：

1. 如果模型返回 reasoning，会合并成 `<think>...</think>` 块。
2. UI 会把思考块折叠为“思考过程”详情区。
3. 一次请求结束后会自动收起思考块。

## 7. 编辑器设置选项说明

设置页中的“编辑器设置”包含：

1. 导出本册：导出当前草稿本 JSON。
2. 组合导出：导出当前草稿本并附带快照。
3. 清空当前页：仅清空当前页行内容，不删除草稿本。
4. 新行首字符为等号：
   1. 默认开。
   2. 开启后：`Enter` 插入新行时，新行首字符为 `=`。
   3. 关闭后：`Enter` 插入空新行（不预填 `=`）。

## 8. 数据与持久化

所有数据默认保存在浏览器 `localStorage`，主要键位：

1. `math-scratch.notebooks.v3`
2. `math-scratch.active-notebook.v3`
3. `math-scratch.recycle-bin.v1`
4. `math-scratch.enter-equation-line-on-enter.v1`
5. `math-scratch.ai-assistant.v1`

说明：

1. 刷新页面不会丢失本地数据。
2. 清浏览器站点数据会丢失本地数据，请定期导出备份。

## 9. 推荐使用流程

1. 新建一个草稿本，按题目或章节拆分页。
2. 逐行推导，阶段性 `Ctrl/Cmd + S` 保存快照。
3. 遇到分支尝试先存快照，再继续改写。
4. 与 AI 协作时优先选 `current-flow` 或 `selected-lines`。
5. 关键阶段导出“组合包”或“全部备份”。

## 10. 常见问题

## Q1：AI 没有参考我的当前推导

排查顺序：

1. 检查上下文来源是否是 `none`。
2. 若是 `selected-lines`，确认已经选中行号。
3. 看设置面板里的上下文摘要是否显示“xx 行”。

## Q2：为什么有时看不到思考过程

1. 需模型实际返回 reasoning 字段才会展示。
2. `off` 模式不会主动请求 thinking。
3. 请求完成后思考块会自动折叠，需要手动展开查看。

## Q3：导入后名字变了

这是预期行为：为避免重名冲突，导入/恢复会自动追加后缀（如“导入”“恢复”）。

## Q4：复制失败

浏览器可能拦截剪贴板权限。请检查站点权限或在 HTTPS/本地开发环境下重试。
